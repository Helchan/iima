# Plugin Core/mpv fidelity evidence

The player-entry realm follows the IINA 1.3.5 Core proxy contract: mutable window, audio, subtitle, and video properties; read-only status; exact relative/absolute/chapter seek routes; history/recent/version values; the reference `formattedTitie` track key; and external video/audio/subtitle track loading. Like IINA reading its own bundle metadata, `core.getVersion()` reports the current replica package identity `0.9.3` and build `93`; IINA `1.3.5` build `141` remains the pinned API/behavior reference. `core.osd()` presents literal plugin text with the reference `From plugin <name>` detail. Its missing-permission exception also retains the reference Info.json declaration guidance verbatim. Window geometry and history use the synchronous, grant-bound plugin bridge. `core.open` resolves `@tmp`, `@data`, track, current-media, home, local, relative, and URL inputs through the reference permission boundary before playback. All non-private/non-track values, including HTTP(S) and relative strings, synchronously require `file-system`; missing current media makes `@current` a logged no-op. The common File/Utils/HTTP-download path preflight also carries the realm role, preventing a global controller (whose reference `player` is nil) from borrowing the child realm's track or current-media resolver.

`iina.mpv` synchronously reads arbitrary live libmpv flags, doubles, strings, and native nodes. Typed writes preserve booleans, strings, non-finite doubles, recursive arrays/maps, and native byte arrays through a tagged JSON-safe representation. Int64 values cross the bridge as decimal text before conversion to JavaScript numbers, and an empty node map retains IINA's nil result. Commands execute synchronously through the owning player's persistent libmpv executor and operation log.

Global-controller realms expose only the reference common surface: `menu`, `standaloneWindow`, `utils`, `file`, `preferences`, `console`, `http`, `ws`, and `global`. Player-only `core`, `mpv`, `event`, `input`, `overlay`, `sidebar`, `playlist`, and `subtitle` objects are created only for entry/child realms.

Automated evidence lives in `scripts/test-plugin-core-mpv.mjs` plus the `mpv`, `player`, and synchronous-dispatch Rust tests. Unlocked third-party plugin compatibility and multi-display geometry remain external acceptance work.
