# Replication Architecture

## Source System

IINA 1.3.5 is a Swift/AppKit application with these important boundaries:

- `PlayerCore.swift`: multi-player lifecycle, playback commands, playlist, thumbnails, screenshot callbacks, plugin loading.
- `MPVController.swift`: direct `libmpv` client API integration, property observation, command execution, render context setup.
- `VideoView.swift` and `ViewLayer.swift`: `CAOpenGLLayer`, `CVDisplayLink`, OpenGL framebuffer rendering, HDR/ICC/EDR handling.
- `MainWindowController.swift`: main player window, OSC, sidebars, full screen, PIP, gestures, drag/drop, OSD.
- `Preference.swift`: UserDefaults-backed preference contract.
- `AppDelegate.RemoteCommandController`, `PlayerCore.NowPlayingInfoManager`, `PlaybackInfo`, and `SleepPreventer`: application-level media commands, Control Center projection, and display-sleep assertion routed through `PlayerCore.lastActive` and all playing cores.
- `TouchBarSupport.swift`, `DurationDisplayTextField.swift`, `PlayerCore.makeTouchBar`, and `MenuController.customTouchBar`: per-player native Touch Bar construction, customization, transport/time/thumbnail presentation, and exact owner routing.
- `MenuController.swift` and menu action files: native menu state, key equivalents, dynamic tracks, chapters, filters, playlist, plugins.
- `JavascriptPlugin*` and `JavascriptAPI*`: plugin runtime, permissions, overlay/sidebar/standalone WebView integrations.
- `FFmpegController.m`: thumbnail generation and media probing through FFmpeg APIs.

## Target System

The Tauri port keeps the same conceptual ownership with different implementation surfaces:

- Frontend (`src/`): exact visual and interaction reproduction of IINA windows and panels.
- Tauri commands (`src-tauri/src/commands.rs`): typed bridge between UI, native dialogs, and native state.
- Player domain (`src-tauri/src/player.rs`): player lifecycle, recent documents, last playback, mpv-style property snapshots, and command semantics backed by one persistent `libmpv` executor per player session.
- Native menu domain (`src-tauri/src/menu.rs`): IINA top-level macOS menu structure, menu accelerators, focused-player routing, and state-derived playlist/chapter/primary-track/second-subtitle submenus.
- mpv contract domain (`src-tauri/src/mpv.rs`):
  - mirrors IINA 1.3.5 `MPVController.observeProperties` as a typed list of 34 observed mpv properties and formats;
  - exposes that contract through Tauri so the current renderer, player reducer, and tests use the same property names as IINA;
  - exposes an IINA-ordered mpv playback-session plan covering client creation, preference-backed option setup, log request, wakeup callback, observed properties, initialization, `vo=libmpv`, and OpenGL render-context setup;
  - probes macOS `libmpv` dylib candidates at runtime with `dlopen` and resolves the client/render symbols used by IINA before enabling the real playback loop;
  - exposes a dynamic client smoke command that creates an mpv client, applies safe startup options, requests warn logs, observes the 34 IINA properties, initializes mpv, and terminates/destroys the client when a dylib is available;
  - owns a persistent executor state object that accepts sequenced player/open/Launch Services/drop/menu `MpvClientOperation` entries, avoids duplicate submissions with a sequence cursor, preserves pending operations while `libmpv` is unavailable, and exposes that status through Tauri commands;
  - creates a long-lived mpv client when `libmpv` is available, sets the app-data `watch-later-directory`, `save-position-on-quit`, and `resume-playback` options before initialization, applies the remaining IINA-shaped startup sequence, observes IINA's 34 properties, initializes mpv, and drains queued `mpv_command` plus typed property writes into the native client API;
  - polls `mpv_wait_event(..., 0)` from executor sync/status calls, mirrors IINA's non-blocking event-drain pattern, decodes `MPV_EVENT_PROPERTY_CHANGE` values for the observed primitive formats, retains a diagnostic tail, and separately hands every newly drained event to the owning player through a lossless queue;
  - registers a real `mpv_set_wakeup_callback` on every client and signals one shared mutex/condition pair without a check-to-sleep race; the awakened backend synchronizer drains the main and every independent player session immediately;
  - decodes structured mpv file lifecycle data for `MPV_EVENT_START_FILE` and `MPV_EVENT_END_FILE`, including playlist entry ids, end-file reason/error, and redirect insertion metadata;
  - applies newly drained core property changes back into `PlayerState` for pause, volume, speed, mute, media title, selected video/audio/subtitle ids, chapter, and idle-active so Tauri command responses are no longer model-only for those properties;
  - applies IINA-ordered lifecycle reducers for `start-file`, `file-loaded`, `playback-restart`, `seek`, `end-file`, and `idle-active`; a non-stop end event while loading records a pending structured error, and only the subsequent idle transition clears the failed player state and exposes the generic open-file alert;
  - actively polls authoritative `libmpv` snapshot properties for path, media-title, duration, time-pos, percent-pos, pause, volume, speed, mute, A-B points/count, audio device/list, video/audio filter Node lists, chapters, playlist counts/position, track count, selected track ids, and idle-active, then reduces them into the Tauri player timeline, filters/devices, playlist current item, and Resume position;
  - polls mpv `track-list/N/*` subproperties into structured track payloads and reduces them into video/audio/subtitle groups carrying selected ids, language, codec, external/default/forced flags, decoder description, dimensions, fps, audio channel, and samplerate metadata;
  - polls the IINA-used mpv `playlist/N/filename`, `playlist/N/current`, `playlist/N/playing`, `playlist/N/title`, and `playlist/N/id` subproperties into structured playlist payloads and reduces them into the Tauri playlist while preserving current/playing state and known durations;
  - is kept warm by the backend wakeup synchronizer even when no web window is visible, with a 250ms timeout for authoritative property polling, while the active frontend uses a visibility-aware 100ms `get_player_snapshot` authority poll guarded against overlap and unchanged repaints; IINA's 10ms/1ms OSC precision adds only a frontend-local 40ms position interpolation ticker;
  - projects the lossless event handoff into an independent per-player 512-record plugin log with a monotonic cursor, emits ordered batches only to that player's WebView, dynamically observes non-core `mpv.<property>.changed` subscriptions as doubles, and lets the frontend cursor deduplicate native wakeups against command snapshots without collapsing repeated native events;
  - now passes its one long-lived client handle into `native_video.m`, which owns an `NSOpenGLView` inside a full-size clear/non-opaque borderless child `NSWindow` ordered below the transparent Tauri parent, clips the outer render container to a continuous 10-point windowed radius, follows the parent once per AppKit main-queue turn during live resize, and performs a full surface/hierarchy/display/mask reconciliation when the drag ends. Fullscreen and screen transitions retain the generation-guarded quiet path, while live exceptions use bounded `50/100/200 ms` recovery without flooding the queue. The view creates an OpenGL `mpv_render_context`, uses a `CVDisplayLink` gate to schedule only mpv-requested/resize-forced redraws, renders the current framebuffer and reports swaps, then frees the context before client teardown;
  - constrains the native player window to mpv's authoritative `dwidth`/`dheight` display aspect, with IINA's two-phase source/user rotation calculation and a `640:360` fallback before video dimensions exist. Fullscreen switches mpv `keepaspect` before the AppKit transition. Renderer attachment is split into prepare and callback phases so synchronous main-thread AppKit work never occurs while the mpv executor mutex is held, and reentrant operations queue until the original attach completes;
  - keeps IINA's 64-bit float, EDR-capable OpenGL surface and maps `display-p3`/`bt.2020` to the same macOS-version-specific Display P3 PQ, ITU-R 2020 PQ, or ITU-R 2100 PQ color spaces. The mapping is applied directly when AppKit supplies a `CAOpenGLLayer`-compatible backing layer and always to the video-only child window, while AppKit's `_NSOpenGLViewBackingLayer` receives the matching EDR-content flag; SDR restores device RGB and the active ColorSync display profile. An EDR source requires the target screen's potential component value to exceed `1.0`; automatic tone-mapping peak discovery dynamically reads CoreDisplay's preferred `ReferencePeakHDRLuminance`, falls back to Catalina's `DisplayBacklight`, then HDR400. Window screen changes, application display-parameter changes, and screen color-profile changes all re-enter this single native color-refresh path without creating another mpv client;
  - Quick Settings now closes the local video/audio/main-subtitle/secondary-subtitle track contract; remaining acceptance is real unusual multi-track media, physical render hardware, and unlocked native lifecycle validation.

Unlocked picker/account/Keychain/Finder interaction, live providers, and physical ICC/HDR hardware remain external validation.
- System media and power boundary (`src-tauri/src/native_system_media.rs`, `src-tauri/src/native_system_media.m`, `src-tauri/src/commands.rs`, `src-tauri/src/lib.rs`):
  - owns one process-level `MPRemoteCommandCenter` registration and routes play/pause/toggle/stop/next/previous/repeat/rate/skip/position callbacks to `AppState.last_active_player_session_label`, the existing typed `PlayerCommand`, and that session's persistent libmpv executor; no native callback keeps a second player model;

Disabling removes every command target and clears `MPNowPlayingInfoCenter`; enabling republishes the last-active projection. Monotonic generations at each native boundary reject delayed background writes after a newer focus, preference, or shutdown decision;
  - projects audio/video type, music title/album/artist or extension-free video title, duration, elapsed time, current/default rate, and unknown/playing/paused/stopped state. Focus changes include session identity in the deduplication key, while the 250ms backend synchronizer compares playing elapsed time against the prior rate-derived clock so ordinary progress does not rewrite the native center and real seeks still do;
  - reduces every live player session to one desired display-sleep state and applies an idempotent `IOPMAssertionCreateWithName(kIOPMAssertionTypeNoDisplaySleep, ...)` / `IOPMAssertionRelease` pair. Closed windows are excluded, multi-session playback holds one assertion until the last playing session pauses/stops/closes, and process exit performs unconditional command/assertion cleanup;

Wheel/trackpad seek and volume use separate reference mappings; seek pauses only a previously playing session and resumes at the gesture boundary, pinch performs centered aspect-preserving resize or directional fullscreen transitions, Force Touch fires once at stage two, and window dragging waits past IINA's strict three-point threshold. Remaining work is unlocked physical mouse/trackpad/Force Touch/window-drag validation.
- Native open panel (`src-tauri/src/native_open_panel.m`, `src-tauri/src/native_open_panel.rs`, `src-tauri/src/commands.rs`):
  - dispatches `NSOpenPanel` to AppKit's main queue and matches IINA's Open File/new-window/Add to Playlist settings: files and directories are selectable together, multiple selection and alias resolution are enabled, and directory creation is disabled;
  - resolves the exact `Choose Media Files` and `Add to playlist` titles through the shared 1.3.5 native localization catalog; the reference multi-open panels do not own accessory views;
  - returns paths as JSON across the Objective-C/Rust boundary so legal path characters do not depend on a delimiter, then shares the same session-aware media transition used by Launch Services batches;
  - both hosts are non-sandboxed, so security-scoped URL ownership is not part of this path. The separate Open URL surface shares the Security.framework HTTP-auth Keychain bridge described above; unlocked native panel modality/selection and real-Keychain interaction remain acceptance work.

## Rendering Decision

The final player cannot use a normal HTML `<video>` element as the playback engine. It would not reproduce mpv filters, scripts, subtitles, configuration files, audio/video track behavior, screenshots, HDR/ICC handling, or mpv property semantics.

The selected rendering path is a native `NSOpenGLView` inside a full-size clear/non-opaque borderless child `NSWindow` ordered below the transparent Tauri parent window. Its outer render container owns the black background and continuous 10-point windowed clip, while fullscreen switches that clip off and window restoration reapplies it. It uses the same libmpv client that receives player commands, matching IINA's `VideoView`/`ViewLayer` ownership boundary: the view owns the GL context, float/EDR surface, EDR layer state, output color-space contract, and render callback, while Rust owns the client lifecycle and state reduction. Parent and child move, resize, enter fullscreen, leave fullscreen, and refresh their display-specific ICC/HDR pipeline as one surface; the web layer contributes only the OSC, sidebars, OSD, and other chrome above the native pixels. This child-window boundary is necessary because an `NSOpenGLView` sibling can occlude WKWebView compositing even when its z-order is nominally below the web content.

Screenshot capture now follows IINA's ownership boundary: the Tauri command resolves the invoking player session, applies `screenshot-directory`, `screenshot-format`, and `screenshot-template` to that session's persistent libmpv client, then issues asynchronous `screenshot subtitles|video` with reply userdata `1000000`. Rust preserves intervening mpv events while awaiting the command reply, selects mpv's newly created file, copies it through macOS `NSPasteboard` when requested, and manages IINA's `screenshot_cache` lifetime for non-saved captures. The web layer only renders the `ScreenshootOSDView`-shaped preview/actions and presents IINA's exact generic failure alert.
