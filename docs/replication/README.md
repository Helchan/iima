# IINA 1.3.5 Tauri Replication

This repository treats `reference/iina` (`reference` is `参考` in the workspace) as the product contract.
The checked source is IINA `release/1.3.5`, commit `45187444`, build `141`.

The Tauri project's release identity is independently versioned as `0.9.3` (build `93`). The
IINA version above remains the behavior/reference baseline. Artifact hashes recorded later in this
document are from the locally rebuilt and verified `0.9.3` macOS package; GitHub Actions rebuilds
tag releases independently.

The goal is not to create a similar media player. The goal is to reproduce IINA 1.3.5 behavior, UI, interaction, integration, and packaging while replacing the implementation stack with Tauri.

## First Principles

- Preserve the product contract: visible UI, menus, shortcuts, preferences, playback behavior, plugin behavior, file associations, URL scheme, CLI, and bundle artifacts must match IINA 1.3.5.
- Preserve the media engine contract: playback remains `libmpv` based. HTML media elements are not an acceptable replacement for IINA playback.
- Preserve native media capabilities: packaged playback and auxiliary probing/thumbnail work share the same dynamically loaded libmpv/FFmpeg runtime without `ffmpeg` or `ffprobe` executable sidecars; screenshots use the active playback client as in IINA.
- Preserve architecture boundaries from IINA where they are still valid:
  - Original `VideoView` and `MPVController` map to a native Tauri media/rendering layer.
  - Original `PlayerCore` maps to the Tauri backend player state and command surface.
  - Original `MainWindowController` and XIB views map to the web UI plus native window controls.
- Do not patch around missing behavior. Every visible difference must be tracked in the acceptance matrix until closed.
